# YouTube Script Writer Skill

When given a topic, write a complete YouTube video script and supporting metadata following the rules below.

**Language: Everything — script, titles, descriptions, chapters — must be in Spanish.** Write in natural spoken [e.g., "Mexican / neutral Latin American"] Spanish. Not formal, not translated-sounding. How the creator actually talks.

---

## Audience

- **Who:** [e.g., "DevOps and platform engineers early in their cloud-native journey"]
- **Knowledge level:** [e.g., "Comfortable with Linux and Docker, unfamiliar with advanced Kubernetes or observability"]
- **Tone:** [e.g., "Conversational and technical — smart but not academic, occasionally dry humor"]

---

## Voice & Style

- **Hook:** [e.g., "Open with a relatable pain point or bold claim. Never 'Hey guys welcome back.'"]
- **Pacing:** [e.g., "Short punchy sentences for emphasis, longer ones for explanation. Vary the rhythm."]
- **Vocabulary:** [e.g., "Real technical terms, defined once on first use. No buzzwords."]
- **Perspective:** [e.g., "First person, present tense. Live-demonstrating, not reading a textbook."]
- **Humor:** [e.g., "Occasional self-deprecating jokes about over-engineered setups. Never forced."]
- **CTAs:** [e.g., "One soft mid-video CTA, one at the end. No begging."]
- **Avoid:** [e.g., "Never open with 'En este video...', no passive voice, no filler transitions"]

## Script Structure

1. **Hook** (0–30s): [e.g., "Lead with the problem or a surprising claim."]
2. **Context** (30–60s): [e.g., "Why this matters — ground it in a real scenario."]
3. **Overview** (60s): [e.g., "One sentence + spoken bullet list of what's covered."]
4. **Main Content**: [e.g., "Each section: heading → explanation → example or demo."]
5. **Recap** (1–2 min): [e.g., "3 key takeaways in plain language."]
6. **Outro** (30s): [e.g., "CTA + optional next video tease."]

---

## Formatting

- Spoken Spanish — contractions and informal phrasing are fine
- `[B-ROLL: descripción]` for visuals or screen recordings
- `[PAUSA]` for natural breath moments
- `**negrita**` for spoken emphasis
- `[DEMO: descripción]` for live demo segments
- Target: **[X] palabras por minuto** (130–150 wpm typical)
