# Directive
- Review the codebase and suggest improvements without touching the code.
- The tone for the review needs to be casual and concise

# Focus on:
- Missing tests
- Security vulnerabilities
- Performance bottlenecks
- Maintainability issues
- Code Readability

# Output
- Provide a report on key areas that you focused on
- Summarize and explain the issues found
